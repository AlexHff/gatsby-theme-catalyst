# Starters

The best way to use Gatsby Theme Catalyst is with starters. The themes require some customization of GraphQL nodes that has to happen at the starter level because of type ownership in Gatsby.

**gatsby-starter-catalyst-core**: Implements `catalyst-core`.

[Demo](https://gatsby-starter-catalyst-core.netlify.com/)

`gatsby new catalyst-core https://github.com/ehowey/gatsby-starter-catalyst-core`

**gatsby-starter-catalyst**: Implements `catalyst-core`, `catalyst-footer`, `catalyst-header-top`.

[Demo](https://gatsby-starter-catalyst.netlify.com/)

`gatsby new catalyst https://github.com/ehowey/gatsby-starter-catalyst`

**gatsby-starter-catalyst-blog**: Implements `catalyst-blog`. The blog theme has a minimal set of styles which is intended as a starting point for further customization.

[Demo](https://gatsby-starter-catalyst-blog.netlify.com/)

`gatsby new catalyst-blog https://github.com/ehowey/gatsby-starter-catalyst-blog`

**gatsby-starter-catalyst-sanity**: Implements `catalyst-sanity`.

[Demo](https://gatsby-starter-catalyst-sanity.netlify.com/)

`gatsby new catalyst-sanity https://github.com/ehowey/gatsby-starter-catalyst-sanity`

**gatsby-starter-catalyst-hydrogen**: Implements `catalyst-hydrogen`. A theme for a freelance writers with data sourced from SANITY.io.

[Demo](https://gatsby-starter-catalyst-hydrogen.netlify.com/)

`gatsby new hydrogen https://github.com/ehowey/gatsby-starter-catalyst-hydrogen`
